## 测试场景

### 场景 1: 本地数据存在时的查询
**状态**: ✅ 已验证（单元测试覆盖）


**预期结果**: 响应时间 < 100ms

### 场景 2: 本地数据不存在，触发采集
**状态**: ⏸️ 待验证（Python 服务需启动）

```
1. 请求基金详情
2. 查询本地数据库 → 不存在

4. 触发 Python 采集
5. 等待采集完成
6. 保存到本地数据库
7. 返回完整数据
```

**预期结果**: 响应时间 < 5s

### 场景 3: 空值缓存生效
**状态**: ⏸️ 待验证（Python 服务需启动）

```
1. 请求不存在的基金
2. 查询本地数据库 → 不存在
3. 调用 Python 采集 → 基金不存在
4. 设置空值缓存（30分钟）
5. 30分钟内再次请求
6. 直接返回空值，不调用 Python
```

**预期结果**: 响应时间 < 50ms

### 场景 4: 并发采集去重
**状态**: ✅ 已验证（单元测试覆盖）

```
1. 10个并发请求同一基金
2. 只触发 1 次 Python 采集
3. 所有请求等待同一结果
4. 返回相同数据
```

**预期结果**: 采集只执行 1 次

### 场景 5: Python 服务异常降级
**状态**: ✅ 已验证（单元测试覆盖）

```
1. 请求基金详情
2. 本地数据不存在
3. 调用 Python 服务 → 异常
4. 降级处理，返回 null
5. 前端显示"数据暂不可用"
```

**预期结果**: 不抛异常，返回友好提示
