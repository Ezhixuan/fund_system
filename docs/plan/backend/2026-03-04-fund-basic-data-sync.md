# 计划: 基金基础数据自动同步机制

**Created**: 2026-03-04 11:30
**Category**: backend
**Status**: Draft
**Priority**: P1

## 背景

系统启动时如果数据库中没有基金基本信息（基金代码、名称等基础数据），会导致：
1. 基金列表查询返回空结果
2. 搜索建议功能无法使用
3. 用户无法知道有哪些基金可以查询

当前仅有单个基金查询具备自动采集机制，列表查询和搜索接口不会触发数据采集。

详见：[Issue: 数据库为空时无法获取基金信息](../../issues/2026-03-04-0228-empty-database-no-fund-data.md)

## 目标

1. Python 端实现获取 MySQL 内基金基础数据条数，并与数据源（akshare）所有基金数量进行比对
2. 数量不一致时，自动更新/补充基金基础数据
3. 服务启动时执行一次数据比对和同步
4. 配置定时任务，每 30 天执行一次全量数据同步检查

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 数据源（akshare）接口变更 | High | 封装数据源访问层，统一异常处理 |
| 全量同步耗时长 | Med | 采用增量同步策略，记录上次同步时间 |
| 定时任务重复执行 | Med | 使用数据库锁或 Redis 分布式锁保证幂等性 |
| 启动时同步阻塞服务启动 | Low | 使用异步线程执行同步，不阻塞主线程 |

## 备注

- 基础数据仅包含基金代码、名称、类型等基本信息，不包含净值、指标等计算数据
- 同步策略：先比对数量，若不一致再比对明细，减少不必要的数据处理
- 建议添加监控告警，当同步失败或数据差异过大时通知运维
