## 一、项目概述

### 1.1 项目目标
构建个人基金交易决策辅助系统，实现数据采集、智能评分、决策信号、持仓管理四大核心能力。

### 1.2 技术架构
- **采集层**：Python 3.11 + akshare + APScheduler
- **业务层**：Java 17 + Spring Boot 3.2 + MyBatis-Plus
- **数据层**：MySQL 8.0 + Redis 7.0
- **前端层**：Vue 3 + ECharts

### 1.3 实施策略
采用**渐进式交付**，分4个Phase迭代开发，每个Phase可独立交付使用。

## 六、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| akshare接口变更 | 高 | Phase 1预留爬虫补偿机制 |

| 前端开发延期 | 中 | Phase 4可先用Swagger UI替代 |
| 信号模型不准 | 中 | 保留调参接口，持续优化 |
